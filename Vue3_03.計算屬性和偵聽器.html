計算屬性和偵聽器

模板內的表達式非常便利，但是設計的初衷是用於簡單運算。在模板中放入太多的邏輯會讓模板過重且難以維護。例如：

{{ author.books.length > 0 ? 'Yes' : 'No' }} ... 太長了，難以閱讀

使用 computed

  • 計算屬性撰寫時是一個函式，但可當成普通屬性綁定於模板
  • 計算屬性會自動偵測是否使用到 data 中的資料，只要使用的資料發生改變，那麼綁定於模板的畫面也會更新。
  • 計算屬性它是一直暫存，只有在資料改變時，它才會重新求值，沒改變就不會動作。
  • 方法則是每次呼叫都會再執行一次，呼叫幾次執行幾次。
